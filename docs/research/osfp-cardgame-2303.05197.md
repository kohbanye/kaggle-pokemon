## 1. TL;DR（なぜこの論文か）

- **探索を一切使わず**（学習時も推論時も look-ahead 探索なし）、**end-to-end の方策+価値ネット**を **OSFP（Optimistic Smooth Fictitious Play）** の自己対戦で学習。`[確度: 高]`
- **デッキ構築（CB: card-building）と対戦（BT: battle）を1つのネットで同時に学習**。カード/ヒーロー埋め込みを両ヘッドで共有。`[確度: 高]`
- Hearthstone のフルゲーム（デッキ構築＋対戦）で、中国トップ10クラスのプロを Best-of-5 で撃破。`[確度: 高]`
- **コンペ適性**: 推論が forward pass 主体＝CPU・時間制約に最適。探索の時間予算管理が不要。**Pokémon TCG の `obs->list[int]`（option index）契約・60枚デッキを init で返す契約**にそのまま対応できる構造。→ 本リポジトリの終着点に採用。

前身研究（同著者・COG2022 二冠）: *"Mastering Strategy Card Game (Legends of Code and Magic) via End-to-End Policy and Optimistic Smooth Fictitious Play"*, arXiv:2303.04096。`[確度: 高]`
本論文はその手法を Hearthstone に適用し、学習安定化の **improved techniques** を積み上げたもの。

## 2. 問題設定（Hearthstone）

- **2人ゼロサム・不完全情報・確率的**なカードゲーム。フルゲームは **(a) デッキ構築 → (b) 対戦** の2段。
- 不完全情報: 相手の手札・山札・引きが見えない。確率性: シャッフル・ランダム効果。
- **ヒーロー（クラス）** が複数あり、デッキ構築は各ヒーロー固有のカードプールから選ぶ。`[確度: 高]`
- 報酬は **勝ち +1 / 負け −1 を試合終了時のみ**（疎・エピソード型）。`[確度: 高]`

> 本コンペとの主な差異は §9 に整理（デッキ枚数 30 vs 60、プール、ヒーロー有無、init の無情報性 など）。

## 3. OSFP（Optimistic Smooth Fictitious Play）— 学習の中核

2人ゼロサムでナッシュ均衡に近づくための **自己対戦メタアルゴリズム**。深層RLを「ベストレスポンス計算器」として包む。

- **Smooth（スムーズ）**: 現方策を **過去チェックポイントの混合** を相手に更新する＝ **Smooth Best Response (SBR)**。最新の1体だけでなく履歴の混合に応答することで、特定相手への過剰適合（循環）を抑える。エントロピー正則化を伴う。`[確度: 高/中]`
- **Optimistic（楽観的）**: 混合の重みを **直近モデルに厚く** する。これにより **last-iterate convergence**（最終反復が均衡に収束）が得られ、**平均方策を別途保持せずに最終チェックポイントをそのまま使える**。`[確度: 高]`（最適化的オンライン学習＝optimistic mirror descent 系の発想。厳密な更新式は原文要確認 `[確度: 中]`）

### Algorithm 1（自己対戦ループの骨子）`[確度: 高/中（パラメータ記号・数値は原文要確認）]`

1. 相手履歴 `H` を空で初期化。
2. 各 **Learning Period (LP)** で、actor は相手を選ぶ:
   - 確率 `p` で **現学習者自身**（self-play）、
   - それ以外は **`H` から勝敗記録に基づく重み付け**でサンプル（直近重視）。
3. LP 内で大量サンプルを集めて RL 更新（§5）。1 LP あたり **≈3.2×10⁸ サンプル**で追加判定。`[確度: 中]`
4. **チェックポイント追加条件**: 現方策が **`H` の全相手を勝率閾値 `ξ` 超で上回る**、**または** `c` 周期連続で未追加なら、現方策を `H` に追加。
5. 収束（last-iterate）した最終チェックポイントを採用。

> 直観: 「過去の自分たち全員に安定して勝てるようになったら殿堂入りさせ、相手プールを更新する」を繰り返す。プールが多様化し、**付け込まれにくい（低 exploitability）戦略**に向かう。

## 4. ネットワーク・アーキテクチャ

- **end-to-end 方策+価値ネット**。1つのネットに **CB ヘッド（デッキ構築）** と **BT ヘッド（対戦）** を持ち、**カード埋め込み・ヒーロー埋め込みを共有**。`[確度: 高]`
- `π_θ ≈ δ_CB·π_θ^CB + δ_BT·π_θ^BT`（局面に応じて使うヘッドが切り替わる、という表現）。`[確度: 中]`
- **再帰**: LSTM（隠れ次元 **256**）で観測履歴を集約 → **不完全情報を明示的な信念状態なしに暗黙学習**。`[確度: 高/中]`
- **価値ヘッド**: CB/BT 双方の中間特徴から1スカラー（勝敗予測）を出す。`[確度: 高]`
- 埋め込み次元など細部は本文に明記されず（Fig.2/付録参照）。`[確度: 中]`

## 5. 行動空間・マスク・RL 学習

### 行動空間（autoregressive 分解）
- 行動を **逐次（autoregressive）** に分解（例: まず種別、次に対象）。これで **各ステップの選択肢を「数十」に抑える**。`[確度: 高]`
- **0/1 合法手マスク**で非合法手の確率をゼロに（softmax 前にマスク）。`[確度: 高]`

### デッキ構築（CB）の具体
- **30枚を 30 ステップで逐次選択**（Hearthstone のデッキは30枚）。各ステップ「プール全カードから1枚選ぶ」カテゴリ分布。`[確度: 高]`
- 観測に `card_set`（全カード）/ `card_selected_mask`（選択済み）/ `card_can_selected_mask`（選択可能）を含め、**選べないカードはマスク**。`[確度: 高]`

### RL アルゴリズム
- **分散 actor-learner**（IMPALA 系）。off-policy 補正に **V-Trace**、勾配に **PPO 損失** を併用。`[確度: 高]`
- **報酬 +1/−1 を試合終了時のみ**、**割引 γ=1.0**（エピソード型ゆえ 0.99→1.0 が効いた）。`[確度: 高]`

## 6. Improved Techniques（学習安定化の積み上げ）

> 効果量は b0 からの累積改善（machine-vs-machine 勝率の改善幅）。**数値は原文要確認** `[確度: 中]`。
> 「どの軸を 1 つずつ足して効いたか」という **ablation の手本**として価値が高い。

| 版 | 変更点 | 効果 | 何を直すか |
|---|---|---|---|
| b0→b1 | **割引 γ=1.0**（vs 0.99） | +7% | 報酬がエピソード末のみ＝割引不要 |
| b1→b1.5 | **Random-CB 初期化**（ランダムデッキで開始） | 同等（勝率維持） | デッキ多様性・人間的プレイを促す |
| b1.5→b2 | **キュー（FIFO）バッファ＋actor半減（c≈1）** | +10% | 生産/消費バランス＝off-policy 誤差低減 |
| b2→b3 | **V-Trace 改良（ρ クリップ）＋ PPO 勾配** | +15.1% | 勾配消失防止・収束安定 |
| b3→b4 | **ヒーロー別モデル（分離）** | +6.5% | ヒーロー固有のプレイスタイルを活用 |
| b4→c5 | **Cheat（相手の一部カードを観測）** | +5.5% | 非対称学習（実運用では使えない実験条件） |

主要なポイント:
- **生産/消費バランス**: 初期は actor が learner より速く（`c>2`）off-policy 誤差が大。**actor を半減**して `c≈1` にし、ring でなく **FIFO キュー**で供給。`[確度: 高/中]`
- **V-Trace のρを下限クリップ**してトレース消失を防止。`[確度: 中]`

## 7. 結果

- **Machine-vs-Machine**（対 b0 ベースライン）: 最良 normal 版 **b4(23日学習) ≈ 73.6%**、cheat 版 **c5(16日) ≈ 80.2%**。`[確度: 中]`
- **Machine-vs-Human**（Best-of-5・フルゲーム＝デッキ構築込み）:
  - b4-23day: **3:0, 3:0**（2大会ストレート勝ち）
  - c5-16day: **3:1, 3:2**（2大会勝ち）`[確度: 中]`
  - 相手は Hearthstone 配信者（中国トップ10クラス）。`[確度: 高]`
- 定性: ドメイン知識なしで **人間的なデッキ**（例: Flamewaker Mage）や **1ターンキル**等の戦術を獲得。`[確度: 中]`

## 9. 本プラン（Pokémon TCG）への落とし込み

### 9.1 そのまま活きる対応
| 論文 | 本コンペ | 反映先 |
|---|---|---|
| end-to-end 方策+価値（探索なし推論） | CPU・10分・手番制限に最適 | [PLAN.md](../../PLAN.md) P3, P5 |
| OSFP 自己対戦（last-iterate・低 exploitability） | Elo ラダー＝多相手環境で付け込まれにくさが効く | P5 |
| CB+BT 同時学習・埋め込み共有 | **デッキを固定せず学習**（init で60枚を返す契約に対応） | P1, P3, P4, P5 |
| autoregressive 行動＋0/1マスク | `obs->list[int]`（option index）契約に直結 | P3 |
| デッキの逐次選択＋合法マスク | 60枚を合法マスク付きで逐次生成 | P1, P3 |
| improved techniques の ablation 手本 | γ/V-Trace/生産消費バランス を1つずつ採否 | P5 |
| 分散 actor-learner / V-Trace+PPO / γ=1.0・±1報酬 | 同構成（Docker sim 並列＋GPU学習） | P5 |

### 9.2 本コンペで論文と違う点（注意）
- **デッキ枚数 30→60、プール ~1,250、ヒーロー無し**: デッキ空間が桁違いに広く、合法性（同名4枚・ACE SPEC≤1・基本ポケモン≥1・60枚ちょうど）が厳しい。→ **合法マスクが必須**（非合法デッキは即敗北）。`[確度: 高（`cg/api.py`・`game.py` で確認）]`
- **init は無情報**: `obs.current`/`obs.select` が共に None＝相手情報ゼロ。デッキは相手に条件付けできず **無条件の分布（混合戦略）** になる。論文のヒーロー条件付き CB とは異なるが、**多相手ラダーで付け込まれにくい混合デッキ**という狙いに合致。`[確度: 高]`
- **探索を“推論時の上乗せ”に拡張**: 論文は探索ゼロだが、本プランは時間予算が余れば **方策=prior／価値=リーフ評価**で決定化浅探索を被せ、**効いた時だけ採用**（P6）。学習ループには入れない（論文の効率/last-iterate を維持）。
- **学習基盤の規模**: 論文 24GPU 級。こちらは GPU 複数台/クラウド予算で **スケールダウン＋BC 暖機**（P4）。
- **cheat は使えない**: 実運用は対称情報。c5 系の手法は不採用。
- ⚠️ **ローカル sim は実ラダーを外す**: 学習デッキ分布の価値（混合 vs 決定的）は特にラダーで検証する（[PLAN.md](../../PLAN.md) §C）。

### 9.3 借用するアイデア（他手法）
- 付け込まれにくさ＝均衡: OSFP で代替し、CFR/DeepNash のフル均衡学習は不採用（発想だけ借用）。
- 確率性の扱い: 必要なら Stochastic MuZero の afterstate 的発想を参照（[game-ai-survey.md](./game-ai-survey.md)）。
