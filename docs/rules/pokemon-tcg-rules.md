# ポケモンカードゲーム ルールリファレンス

> Pokémon TCG AI Battle（シミュレーショントラック）のエージェント実装の土台として、
> ポケモンカードゲームの基本ルールを整理したもの。
> ルールは**実物のポケモンカードゲーム（剣盾〜SV期の現行ルール）**に準拠し、
> 本リポジトリのデータ表現（`src/cards.py`）やエージェント契約（`agent(obs) -> list[int]`）との
> 対応も併記する。最終的な挙動は **同梱シミュレータ（`cg/`）の実装が正**なので、
> 細部はsim挙動で確認すること。

---

## 0. 30秒サマリ（AI視点）

- 2人対戦。**先に「サイド6枚を取り切る」「相手の場のポケモンを全滅させる」「相手の山札切れ」のいずれか**を満たした側が勝ち。
- 自分のターンは「**ドロー → 各種アクション（任意の順・回数制限あり）→ ワザで攻撃してターン終了**」。
- 各ターンにできる主な行動は **エネルギー手貼り1回 / サポート1回 / にげる1回** の回数制限がある。グッズや進化・ベンチ展開は条件内なら複数回可。
- 攻撃すると基本ターン終了。**相手のバトルポケモンをきぜつさせるとサイドを取れる**（exなら2枚など）。
- 不完全情報ゲーム：相手の手札・山札・サイドの中身は見えない（探索では決定化が必要 → `docs/research/game-ai-survey.md`）。

---

## 1. 勝利条件（いずれか先に達成で勝ち）

1. **サイドを取り切る** — 自分のサイド（最初6枚）を全部取った。
2. **相手の場のポケモン全滅** — 相手のターン開始時などに、相手の場（バトル＋ベンチ）にポケモンが1匹もいない。
3. **相手の山札切れ（デッキアウト）** — 相手が自分のターン最初のドローで山札からカードを引けない。

> 引き分け扱い・同時達成の優先順位など細部はsim実装に従う。

---

## 2. ゲームの構成要素

### 2.1 デッキ構築ルール
- **ちょうど60枚**。
- **同名カードは4枚まで**（基本エネルギーは枚数無制限）。
- **ACE SPEC** はデッキに**合計1枚だけ**（後述）。
- たねポケモンを最低1枚は入れる（入っていないと初期手札を引き直し続ける＝事実上必須）。

> エージェント契約上、初期選択（`obs.select is None`）では **60枚のカードIDリストを返す**（= デッキ提出）。
> `deck.csv` がデッキ定義、`main.py` がエージェント本体。

### 2.2 場（フィールド）の区画
- **バトル場**：ワザを使う前線のポケモン1匹。
- **ベンチ**：控え。**最大5匹**。
- **山札（デッキ）**／**手札**／**トラッシュ（捨て札）**／**サイド（6枚、裏向き）**。
- **スタジアム**：場に1枚だけ存在できる共有カード。

### 2.3 初期セットアップ
1. デッキをシャッフルし、**手札7枚**を引く。
2. たねポケモンが手札に無ければ公開してマリガン（引き直し、相手は枚数に応じ追加ドロー）。
3. バトル場に**たねポケモン1匹**、ベンチに任意でたねを置く。
4. **サイドを6枚**裏向きで置く。
5. じゃんけん等で先攻後攻を決める。**先攻の最初のターンはワザを使えない**（sim実装に従う）。

## 3. カードの種類

`src/cards.py` の `supertype()` は stage/type 列から **Pokemon / Trainer / Energy** に大別する。

### 3.1 ポケモン
- **進化段階**（`stage_or_type`）：`Basic Pokémon`（たね）/ `Stage 1 Pokémon`（1進化）/ `Stage 2 Pokémon`（2進化）。
- 持つ情報：**HP**、**タイプ**（`type_code`）、**弱点**（`weakness_code`）、**抵抗力**、**にげるコスト**（`retreat`）、**ワザ／特性**。
- **進化**：場のポケモンの上に対応する進化カードを重ねる。
  - 進化は **そのポケモンを出した／前回進化させたターンは不可**（場に出たターンは進化できない）。
  - 進化すると **状態異常は回復**、ワザの効果ターン跨ぎなどはリセット（細部はsim）。
- **特別なポケモン**（`src/cards.py` の `rule` 列で判定）：
  - **ex**（`is_ex`）：強力な代わりに**きぜつさせられるとサイドを2枚**相手に取られる。
  - **Mega（メガ）**（`is_mega`）：メガシンカ系。きぜつでサイド複数枚（ルール文に従う）。
  - 一般に「ルールを持つポケモン」（ex等）は倒されたときのサイド枚数が多い＝ハイリスク・ハイリターン。

### 3.2 トレーナーズ（`_TRAINER_STAGES`）
- **グッズ（Item）**：1ターンに**何枚でも**使える使い切りカード。
- **サポート（Supporter）**：1ターンに**1枚だけ**。強力なドロー／妨害が多い。
- **ポケモンのどうぐ（Pokémon Tool）**：ポケモンに付ける。1匹に原則1個。
- **スタジアム（Stadium）**：場に1枚。新しいスタジアムを出すと既存のものはトラッシュ。同名は重ね出し不可。

### 3.3 エネルギー（`_ENERGY_STAGES`）
- **基本エネルギー（Basic Energy）**：デッキ枚数無制限。
- **特殊エネルギー（Special Energy）**：効果付き。同名4枚制限。
- **エネルギーは1ターンに手札から1枚しか手貼りできない**（カード効果による追加付与は別）。

#### エネルギー記号の対応（`ENERGY_SYMBOLS`）
| 記号 | タイプ | 記号 | タイプ |
|---|---|---|---|
| `G` | Grass（草） | `F` | Fighting（闘） |
| `R` | Fire（炎） | `D` | Darkness（悪） |
| `W` | Water（水） | `M` | Metal（鋼） |
| `L` | Lightning（雷） | `N` | Dragon（竜） |
| `P` | Psychic（超） | `A` | Special（特殊扱い） |
| `C` | Colorless（無色） | | |

- **ワザコスト表記**：`{G}` 等は色指定エネルギー、`●`（colorless）は**どのエネルギーでもよい1個**。
  `parse_cost("{D}●●") -> {"D": 1, "C": 2}`（`●` は `"C"` に集計）。
- **`C`（無色）コスト**は手持ちの任意のエネルギーで支払える。

## 4. ターンの流れ

自分のターンは概ね次の順：

1. **ドロー**：山札から1枚引く（引けなければ負け＝デッキアウト）。
2. **メインフェイズ（任意の順・回数制限内で自由に）**：
   - 手札からたねポケモンを**ベンチに出す**（空きがあれば複数可）。
   - 場のポケモンを**進化**させる（各ポケモンにつき条件内で）。
   - **エネルギーを手貼り**（**1ターン1回**）。
   - **グッズ**を使う（複数可）／**サポート**を使う（**1ターン1枚**）／**スタジアム**を出す。
   - **どうぐ**を付ける、**特性**を使う（特性は回数条件がカードごとに異なる）。
   - **にげる**（**1ターン1回**、にげるコスト分のエネルギーをトラッシュしてベンチと入れ替え）。
3. **ワザを使って攻撃 → ターン終了**。
   - ワザのコストを満たすエネルギーがバトルポケモンに付いていれば宣言。
   - ワザを使うと（一部例外を除き）そのターンは終了。
   - 攻撃しないでターンを終える選択も可。

> **先攻1ターン目はワザ不可**（多くのルール／sim）。エネ手貼り・サポート使用などの制限もsimの`obs`提示する合法手に従うのが安全。

## 5. 戦闘の解決

### 5.1 ダメージ計算
1. ワザの**基本ダメージ**（`damage`、`parse_damage` は数値部分を抽出。`30×`→30, `120`→120）。
2. **弱点**：相手の弱点タイプで攻撃すると**ダメージ2倍**（×2が現行主流。sim実装に従う）。
3. **抵抗力**：相手の抵抗力タイプなら**−20等**軽減。
4. 各種**効果による増減**（`+`, `-`, `×` 修飾、`effect` 列の文章効果）。
5. HP以上のダメージが乗ると**きぜつ**。

> `parse_damage` は修飾（`×`, `+`, `-`）を無視した**基本値**しか返さない。
> 実際の打点計算は弱点・抵抗・効果を sim 側で解決する必要がある。

### 5.2 きぜつとサイド
- バトルポケモンがきぜつ → そのカードと付属（エネ・どうぐ）を**トラッシュ**。
- きぜつさせた側は**サイドを引く**：通常**1枚**、**ex は2枚**（Mega等はルール文に従う）。
- バトル場が空いたら、きぜつした側は**ベンチから新しいバトルポケモンを出す**（出せなければ負け）。

### 5.3 状態異常（バトル場のみ）
- **どく（Poisoned）**：ポケモンチェックで10ダメージ等。
- **やけど（Burned）**：ダメージ＋コインで継続判定。
- **ねむり（Asleep）**：ワザ・にげる不可、コインで回復判定。
- **マヒ（Paralyzed）**：ワザ・にげる不可、次の自分のターンに回復。
- **こんらん（Confused）**：ワザ使用時コイン、失敗で自分にダメージ。
- 状態異常は **進化／にげる／ベンチに下げる** などで回復（細部はsim）。

## 6. このコンペ特有の注意点

- **エージェント契約**：`agent(obs_dict) -> list[int]`。
  - 初期選択（`obs.select is None`）→ **60枚のデッキ（カードIDリスト）**を返す。
  - 以降は `obs` が提示する**選択肢（options）のインデックス**を返す。
  - **絶対にクラッシュしない**（例外時は合法手フォールバック）。**手番／試合の時間制限**を超えない。
- **合法手は sim が `obs` で提示する**ので、ルールの細則を自前で完全再現するより
  「提示された options から選ぶ」のが安全。ルール知識は**評価・先読みの質**に効く。
- **不完全情報**：相手手札・山札・サイドは不可視。探索は決定化（PIMC/ISMCTS）が必要
  → `docs/research/game-ai-survey.md` 参照。
- **ACE SPEC**（`is_ace_spec`）：デッキに1枚制限の強力カード。デッキ構築の制約として要考慮。
- **エンジンAPI**：`all_card_data()` / `all_attack()` で全カード・ワザデータ、
  `search_begin` / `search_step` で先読み探索が可能（`scripts/sim_smoke.py` 参照）。

## 7. AI設計に効くルール上のレバー（メモ）

- **サイドレース**が勝敗の本質：取る／取らせる枚数の差分（特にexの2枚）を評価関数の主軸に。
- **盤面テンポ**：エネ手貼りは1ターン1回が基本 → エネ加速・進化の前借りが大きなアドバンテージ。
- **きぜつ／被きぜつのリスク**：自分のexが倒されると一気にサイド2枚 → 出し方・退かせ方が重要。
- **手札リソース**：サポート1枚／ターンの制約下で、ドロー・サーチの質が回し続けられるか。
- **弱点**：×2は計算が壊れるほど大きい → デッキ相性・対面評価の主要因。
